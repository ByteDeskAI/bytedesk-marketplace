Smells

Anemic Facade

smell / boundary-smell

A facade merely forwards calls while leaking subsystem concepts, errors, configuration, and lifecycle rules to callers.

Chatty Integration

smell / integration-smell

A workflow performs many small cross-boundary calls where one coarse contract, document message, or asynchronous exchange would be clearer.

Conditional Sprawl

smell / object-smell

Repeated branches choose behavior, type construction, lifecycle state, or command handling across multiple modules.

Event Soup

smell / integration-smell

Many events circulate without clear ownership, naming discipline, contract governance, or traceable consumer responsibility.

Fragile Adapter

smell / boundary-smell

An adapter hides a dependency name but leaks payload shape, protocol quirks, exceptions, or retry behavior.

God Service

smell / object-smell

One service coordinates unrelated responsibilities, owns too much state, and becomes the default place for new behavior.

Hidden Router

smell / integration-smell

Routing decisions are scattered across producers, consumers, configuration, and conditionals with no single testable policy.

Lifecycle State Spread

smell / object-smell

State checks and transition rules are scattered across services, controllers, handlers, or UI logic.

Naive Exactly Once

smell / integration-smell

The design assumes messages, commands, or jobs execute exactly once without durable idempotency, transaction boundaries, or recovery policy.

Overused Singleton

smell / object-smell

Global instances are used for convenience, configuration, service lookup, or mutable process-wide state.

Pattern Ceremony

smell / design-smell

Pattern-shaped classes, interfaces, or layers exist before the design has enough variation, ownership pressure, or runtime risk to justify them.

Provider Switch Sprawl

smell / boundary-smell

Provider, engine, tenant, or region selection is repeated across services, handlers, or UI code.

Unbounded Retry

smell / integration-smell

Failed work retries indefinitely or with unclear terminal behavior, often without idempotency or operator visibility.
